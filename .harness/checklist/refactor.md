# 重构检查清单

## 准备阶段
- [ ] 明确重构目标与范围
- [ ] 评估重构风险
- [ ] 确认现有测试覆盖
- [ ] 制定回滚方案
- [ ] 获取利益相关者同意

## 分析阶段
- [ ] 识别代码坏味道
- [ ] 分析依赖关系
- [ ] 评估技术债务
- [ ] 确定重构优先级
- [ ] 制定重构计划

## 执行阶段
- [ ] 按计划逐步重构
- [ ] 保持测试通过
- [ ] 避免引入新功能
- [ ] 提交粒度适中
- [ ] 代码审查及时

## 验证阶段
- [ ] 所有测试通过
- [ ] 功能行为一致
- [ ] 性能无退化
- [ ] 代码质量提升
- [ ] 文档已更新

## 完成阶段
- [ ] 总结重构成果
- [ ] 更新技术文档
- [ ] 分享经验教训
- [ ] 确认无遗留问题
