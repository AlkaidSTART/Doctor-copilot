# Bug修复检查清单

## 问题分析
- [ ] 确认 Bug 复现步骤
- [ ] 确认影响范围（用户群体、数据）
- [ ] 确认严重程度（P0-P3）
- [ ] 确认是否有数据损坏风险
- [ ] 记录问题日志与截图

## 根因定位
- [ ] 分析相关代码路径
- [ ] 确认问题根源（前端/后端/数据库/配置）
- [ ] 确认是否为回归问题
- [ ] 确认是否有类似问题存在

## 修复方案
- [ ] 制定修复方案
- [ ] 评估修复影响范围
- [ ] 确认修复不会引入新问题
- [ ] 制定回滚方案

## 开发与测试
- [ ] 编写修复代码
- [ ] 编写回归测试用例
- [ ] 本地验证修复效果
- [ ] 测试边缘场景
- [ ] 确认数据一致性（如有数据变更）

## 发布
- [ ] 代码审查通过
- [ ] CI/CD 构建通过
- [ ] 灰度发布验证（如适用）
- [ ] 完整发布并监控
- [ ] 记录变更日志

## 事后分析
- [ ] 记录问题根因
- [ ] 总结经验教训
- [ ] 提出预防措施
- [ ] 更新相关文档
