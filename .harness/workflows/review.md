# 代码审查流程

## 触发条件
- 开发完成，提交 Pull Request
- 代码变更影响核心模块
- 安全相关代码变更
- 架构相关代码变更

## 流程步骤

### 1. 提交审查
- 创建 Pull Request
- 填写 PR 描述
- 指定审查人员
- 关联相关 Issue/Ticket

### 2. 初步检查
- 检查 CI/CD 状态
- 检查测试覆盖率
- 检查代码格式
- 检查变更范围

### 3. 深入审查
- 审查代码逻辑
- 审查安全问题
- 审查性能问题
- 审查可维护性
- 审查测试用例

### 4. 反馈与修正
- 提出审查意见
- 开发者修正问题
- 审查人员确认修正
- 必要时重新审查

### 5. 合并
- 所有审查通过
- CI/CD 构建成功
- 合并到目标分支
- 删除临时分支

## 审查标准
- 代码质量（编码规范、可读性）
- 功能正确性（需求实现、边界处理）
- 安全性（注入攻击、权限控制）
- 性能（查询优化、资源使用）
- 测试（覆盖率、用例合理性）

## 交付物
- 审查通过的代码
- 审查意见记录
- 测试报告

## 参与角色
- 开发者
- 代码审查人员
- 技术负责人（如适用）
