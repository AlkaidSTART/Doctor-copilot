# 数据库规则

## 数据库设计

### 设计原则
- 使用规范化设计（3NF）
- 使用 UUID 作为主键
- 使用 timestamptz 存储时间
- 使用 jsonb 存储非结构化数据

### 表结构规范
- 表名使用 snake_case
- 字段名使用 snake_case
- 主键命名为 id
- 外键命名为 {table_name}_id

### 索引策略
- 为主键创建索引
- 为外键创建索引
- 为常用查询字段创建复合索引
- 避免过度索引

### 约束规范
- 使用 NOT NULL 约束必填字段
- 使用 UNIQUE 约束唯一字段
- 使用 CHECK 约束字段值范围
- 使用 FOREIGN KEY 约束关联关系

## 数据访问

### ORM 使用
- 使用 Supabase Client 进行数据访问
- 使用 Prisma（如需要）作为 ORM
- 使用参数化查询防止 SQL 注入

### 查询优化
- 使用 EXPLAIN 分析查询计划
- 避免全表扫描
- 使用覆盖索引
- 限制查询结果数量

### 事务管理
- 使用数据库事务保证数据一致性
- 使用 BEGIN/COMMIT/ROLLBACK
- 避免长事务

## RLS（行级安全性）

### RLS 策略
- 所有表默认启用 RLS
- 使用 org_id + role_claim 作为策略核心
- 策略规则：
  - 用户只能访问所属机构的数据
  - 用户只能访问有权限的数据
  - 管理员可以访问所有数据

### RLS 测试
- 测试不同角色的访问权限
- 测试跨机构访问控制
- 测试权限变更后的访问控制

## 数据迁移

### 迁移工具
- 使用 Supabase CLI 进行迁移

- 迁移脚本版本控制

### 迁移规范
- 迁移脚本按顺序编号
- 迁移脚本幂等性
- 迁移前备份数据
- 迁移后验证数据

### 回滚策略
- 准备回滚脚本
- 测试回滚流程
- 记录迁移日志

## 数据安全

### 敏感数据
- 敏感数据加密存储
- 使用 PostgreSQL pgcrypto 扩展
- 敏感数据脱敏输出

### 访问控制
- 使用 RBAC 控制数据库访问
- 使用 RLS 控制行级访问
- 使用最小权限原则

### 审计日志
- 记录数据变更日志
- 包含操作人、操作时间、操作内容
- 审计日志不可篡改

### 查询优化

### 数据库配置

### 数据归档
