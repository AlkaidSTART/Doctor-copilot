# 安全规则

## 认证安全

### 认证方式
- 使用 Supabase Auth 进行身份验证
- 使用 JWT Token 进行无状态认证
- 使用 Session Cookie 进行有状态认证
- 使用 HTTPS 传输认证信息

### Token 管理
- Token 过期时间合理（15-60 分钟）
- 使用 Refresh Token 获取新 Token
- Token 存储在 HttpOnly Cookie 中
- 避免将 Token 存储在 localStorage

### 认证流程
- 登录：验证用户名密码，返回 Token
- 请求：携带 Token，验证 Token 有效性
- 退出：清除 Token，使 Token 失效

## 授权安全

### RBAC（基于角色的访问控制）
- 定义角色（doctor、nurse、admin、patient）
- 定义权限（read、write、delete）
- 角色分配权限
- 用户分配角色

### RLS（行级安全性）
- 所有表启用 RLS
- 使用 org_id + role_claim 作为策略核心
- 用户只能访问所属机构的数据
- 用户只能访问有权限的数据

### 最小权限原则
- 只授予必要的权限
- 定期审查权限
- 及时撤销不必要的权限

## 输入验证

### 验证策略
- 使用 Zod Schema 验证所有输入
- 验证参数类型、长度、格式
- 验证业务规则
- 返回详细的错误信息

### 防止注入攻击
- 使用参数化查询
- 避免字符串拼接 SQL
- 使用 ORM 框架

### 防止 XSS 攻击
- 转义危险 HTML
- 使用安全的渲染方法
- 设置 CSP 头

### 防止 CSRF 攻击
- 使用 SameSite Cookie
- 使用 CSRF Token
- 验证请求来源

## 数据安全

### 敏感数据保护
- 敏感数据加密存储
- 使用 PostgreSQL pgcrypto 扩展
- 敏感数据脱敏输出
- 避免在日志中记录敏感数据

### 数据访问控制
- 使用 RLS 控制行级访问
- 使用 RBAC 控制资源访问
- 使用最小权限原则

### 数据备份
- 定期备份数据
- 备份存储在安全位置
- 加密备份数据
- 测试备份恢复

## 传输安全

### HTTPS
- 所有请求使用 HTTPS
- 使用有效的 SSL 证书
- 强制 HTTPS 重定向

### TLS 配置
- 使用 TLS 1.2+
- 禁用弱密码套件
- 配置 HSTS 头

### CORS 配置
- 限制允许的来源
- 限制允许的方法
- 限制允许的头部

## 服务器安全

### 访问控制
- 使用防火墙限制访问
- 使用安全组配置
- 定期审查访问日志

### 安全更新
- 定期更新系统和依赖
- 及时修复安全漏洞
- 使用 Dependabot 自动更新

### 密钥管理
- 使用环境变量存储密钥
- 使用密钥管理服务
- 定期轮换密钥
- 避免硬编码密钥

## 日志安全

### 日志内容
- 避免记录敏感信息
- 记录必要的上下文信息
- 使用结构化日志

### 日志访问
- 限制日志访问权限
- 定期审查日志
- 日志加密存储
