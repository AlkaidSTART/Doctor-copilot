# 代码质量规则

## 代码规范

### 编码标准
- 遵循 TypeScript 编码规范
- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化

### 命名规范
- 文件命名：kebab-case
- 变量命名：camelCase
- 函数命名：camelCase
- 类/接口命名：PascalCase
- 常量命名：UPPER_SNAKE_CASE

### 注释规范
- 使用 JSDoc 注释
- 注释说明代码意图
- 避免冗余注释
- 注释保持更新

## 代码结构

### 文件结构
- 文件大小不超过 200 行
- 函数大小不超过 50 行
- 类大小不超过 100 行
- 模块职责清晰

### 代码组织
- 按功能域组织代码
- 相关代码放在一起
- 使用目录结构分层
- 避免平铺文件

### 导入规范
- 使用绝对路径导入
- 导入顺序：外部依赖、内部依赖、相对路径
- 使用 import type 导入类型
- 避免循环导入

## 错误处理

### 错误分层
- 业务错误：业务规则验证失败
- 系统错误：基础设施错误
- 认证错误：权限不足

### 错误捕获
- 使用 try-catch 捕获错误
- 在边界层处理错误
- 返回统一错误格式

### 错误日志
- 记录错误详情
- 包含请求上下文
- 使用结构化日志

## 类型安全

### TypeScript 使用
- 使用严格模式
- 使用类型定义
- 避免 any 类型
- 使用类型断言谨慎

### 类型检查
- 运行 TypeScript 编译检查
- 使用 ESLint 类型规则
- 使用类型守卫

### 类型定义
- 定义共享类型
- 使用泛型
- 使用类型别名

## 测试规范

### 测试覆盖
- 单元测试：80%+
- 集成测试：60%+
- E2E 测试：覆盖核心流程

### 测试策略
- 测试边界条件
- 测试异常情况
- 测试性能指标

### 测试工具
- Jest：单元测试
- React Testing Library：组件测试
- Cypress：E2E 测试

## 性能优化

### 代码优化
- 避免不必要的计算
- 使用缓存
- 使用懒加载
- 使用代码分割

### 数据库优化
- 使用索引
- 避免 N+1 查询
- 使用批量操作
- 使用连接池

### 网络优化
- 合并请求
- 使用缓存
- 使用压缩

## 安全规范

### 输入验证
- 使用 Zod Schema 验证
- 防止注入攻击
- 防止 XSS 攻击
- 防止 CSRF 攻击

### 输出过滤
- 移除敏感信息
- 数据脱敏
- 防止数据泄露

### 访问控制
- 使用 RBAC
- 使用 RLS
- 最小权限原则

### 审查标准

- 安全性
- 性能
- 可维护性

### 审查流程
- 提交 PR
- 代码审查
- 反馈修正
- 合并代码

### 审查工具
- GitHub Pull Request
- ESLint
- TypeScript Compiler
