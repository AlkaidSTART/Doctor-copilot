# 编码风格规则

## 文件格式

### 文件编码
- 使用 UTF-8 编码
- 文件末尾添加换行符
- 避免 BOM 头

### 文件大小
- 单个文件不超过 200 行
- 超过 200 行考虑拆分

### 文件命名
- 使用 kebab-case
- 使用小写字母
- 使用连字符分隔单词
- 避免使用下划线

## 代码格式

### 缩进
- 使用 2 个空格缩进
- 避免使用制表符

### 空格
- 运算符前后添加空格
- 逗号后添加空格
- 花括号前后添加空格
- 函数参数后添加空格

### 换行
- 一行不超过 120 个字符
- 适当换行提高可读性
- 避免超长行

### 括号
- 使用花括号包裹代码块
- if/else 语句使用花括号
- 函数参数使用括号

## 命名规范

### 变量命名
- 使用 camelCase
- 使用有意义的名称
- 避免使用缩写
- 避免使用单字母变量（循环变量除外）

### 函数命名
- 使用 camelCase
- 使用动词开头
- 使用有意义的名称
- 避免使用缩写

### 类/接口命名
- 使用 PascalCase
- 使用名词或名词短语
- 使用有意义的名称

### 常量命名
- 使用 UPPER_SNAKE_CASE
- 使用有意义的名称
- 避免使用魔法数字

### 类型命名
- 使用 PascalCase
- 使用 Type 后缀（如 UserType）
- 使用有意义的名称

## 注释规范

### 注释类型
- JSDoc 注释：用于函数、类、接口
- 行注释：用于代码解释
- 块注释：用于复杂逻辑说明

### JSDoc 注释
- 使用 /** ... */ 格式
- 包含参数说明
- 包含返回值说明
- 包含示例（如需要）

### 行注释
- 使用 // 格式
- 注释在代码上方
- 注释说明代码意图
- 避免冗余注释

### 块注释
- 使用 /* ... */ 格式
- 用于复杂逻辑说明
- 用于临时禁用代码
- 避免滥用

## 代码组织

### 导入顺序
- 外部依赖（npm 包）
- 内部依赖（packages）
- 相对路径依赖
- 类型导入（import type）

### 代码顺序
- 变量声明
- 函数定义
- 类定义
- 导出语句

### 函数顺序
- 公共方法在前
- 私有方法在后
- 辅助方法在最后

## 条件语句

### if/else
- 使用花括号
- 条件表达式清晰
- 避免嵌套过深（不超过 3 层）
- 使用早期返回

### switch
- 使用 break
- 使用 default
- 避免过长的 switch

### 三元运算符
- 用于简单条件
- 避免复杂嵌套
- 使用括号提高可读性

## 循环语句

### for 循环
- 使用 let/const 声明循环变量
- 避免无限循环
- 使用 break/continue 控制流程

### forEach/map/filter
- 优先使用数组方法
- 避免在循环中修改数组
- 使用箭头函数

## 函数编写

### 函数长度
- 单个函数不超过 50 行
- 超过 50 行考虑拆分

### 函数参数
- 参数不超过 3 个
- 使用对象传递多个参数
- 使用默认参数

### 函数返回
- 使用早期返回
- 返回类型明确
- 避免返回 undefined

### 箭头函数
- 用于简单函数
- 用于回调函数
- 避免过长的箭头函数

## 类编写

### 类结构
- 构造函数
- 公共方法
- 私有方法
- 静态方法

### 类属性
- 使用 private/public/protected
- 使用 readonly
- 避免使用 public 属性

### 类方法
- 使用 camelCase
- 使用有意义的名称
- 避免过长的方法

## 错误处理

### try-catch
- 在边界层使用
- 捕获特定错误
- 记录错误日志
- 返回统一错误格式

### 错误类型
- 使用自定义错误类
- 区分业务错误和系统错误
- 使用错误码

## 类型安全

### TypeScript 使用
- 使用严格模式
- 使用类型定义
- 避免 any 类型
- 使用类型断言谨慎

### 类型声明
- 使用 interface 声明对象类型
- 使用 type 声明联合类型
- 使用泛型提高复用性

## 代码审查检查

### 格式检查
- 缩进正确
- 空格正确
- 换行正确
- 命名规范

### 质量检查
- 代码可读性
- 代码复杂度
- 类型安全
- 错误处理
