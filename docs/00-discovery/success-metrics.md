# Success Metrics

## 背景

MVP 需要同时评估产品可用性、临床流程价值与 AI 效果。

## 为什么

无指标即无法验证产品假设是否成立。

## 目标

建立可执行、可追踪的指标体系。

## 非目标

- 不包含长期财务指标预测。

## 范围

上线后 1~3 个月可观测指标。

## 流程图（Mermaid）

```mermaid
flowchart LR
  A[产品使用] --> B[流程效率]
  B --> C[临床结果代理指标]
  C --> D[持续优化]
```

## ASCII 图

```text
Adoption -> Efficiency -> Safety -> Retention
```

## 表格

| 类别    | 指标                  | 目标值（MVP） |
| ------- | --------------------- | ------------- |
| 采用    | 周活医生占比          | >= 70%        |
| 执行    | 随访任务完成率        | >= 65%        |
| 风险    | 高风险告警 2h 响应率  | >= 80%        |
| AI 质量 | 医生采纳 Brief 建议率 | >= 40%        |
| 稳定性  | 核心接口可用性        | >= 99.9%      |

## 示例

若“Brief 建议采纳率”连续两周低于 30%，触发 Prompt 与检索策略回归评估。

## 风险

| 风险           | 缓解                   |
| -------------- | ---------------------- |
| 指标可被“刷高” | 联合过程指标与结果指标 |
