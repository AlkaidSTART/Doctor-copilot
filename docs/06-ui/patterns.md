# Patterns 交互模式

## 概述

交互模式是跨组件、跨页面的通用行为约定。本文档定义 Doctor Copilot 中高频出现的交互模式，确保用户在不同场景下获得一致的操作体验。

## 抽屉 / 侧滑面板

抽屉用于在不离开当前页面的情况下展示补充信息或执行轻量操作。

### 使用场景

- 从 Dashboard 查看患者最新 Timeline 事件
- 从任务列表快速编辑任务状态
- 从消息中心预览消息详情

### 决策树

```text
信息量少 + 不离开当前页  →  Drawer
信息量大 + 需要完整编辑  →  独立页面
需要用户确认 / 阻断操作   →  Modal
```

### 抽屉结构

```text
┌─────────────────────────────────┐
│ [返回] 标题              [×]    │  ← Header，高度 64px
├─────────────────────────────────┤
│                                 │
│           内容区                 │  ← Padding 20px
│                                 │
├─────────────────────────────────┤
│ [取消]             [主要操作]    │  ← Footer，可选
└─────────────────────────────────┘
```

### 移动端适配

- 抽屉从底部滑出，占满屏幕
- 顶部有 36px 拖动条，可下滑关闭
- 内容过长时内部滚动，不影响背景页

### 行为约定

- 打开抽屉时背景页不滚动
- 抽屉内表单未保存时关闭需二次确认
- 复杂详情抽屉提供「查看完整页面」入口

## 筛选与搜索

### 全局搜索

- 快捷键：`Cmd/Ctrl + K`
- 搜索面板居中显示，宽度 `560px`
- 支持搜索患者、任务、文档、帮助
- 搜索结果分类展示，高亮匹配关键字

```text
┌─────────────────────────────────┐
│ 🔍 搜索患者、任务、文档...       │
├─────────────────────────────────┤
│ 患者                            │
│  ├─ 张三（高血压随访）           │
│  └─ 李四（术后复查）             │
│ 任务                            │
│  └─ 今日待完成随访 3 项          │
└─────────────────────────────────┘
```

### 页面筛选

- 筛选控件位于列表/表格上方
- 常用筛选：日期范围、状态、风险等级、负责人
- 筛选结果实时更新，URL 同步记录筛选条件
- 已选筛选项以 Tag 形式展示，可单独移除

```text
┌─────────────────────────────────────────────────────────┐
│ [全部状态 ▼] [P1-P4 ▼] [最近 7 天 ▼]  [搜索...] [筛选]  │
│ 已选：P1 [×]  最近 7 天 [×]                              │
└─────────────────────────────────────────────────────────┘
```

### 筛选状态

| 状态 | 表现 |
|---|---|
| 无筛选 | 展示占位提示「筛选患者、状态...」 |
| 已筛选 | 展示 Tag，表格上方显示结果数量 |
| 筛选无结果 | 展示空状态，提供清除筛选入口 |

## 表单模式

### 表单布局

- 单列布局为主，字段宽度与内容类型匹配
- 标签左对齐或顶对齐，医疗长表单推荐顶对齐
- 字段间距：`24px`
- 表单操作按钮固定在底部或卡片 Footer

```text
┌─────────────────────────────┐
│ 患者姓名                    │
│ [____________________]      │
│                             │
│ 随访类型                    │
│ [__________ ▼]              │
│                             │
│ 备注                        │
│ [                        ]  │
│ [                        ]  │
├─────────────────────────────┤
│ [取消]         [保存]       │
└─────────────────────────────┘
```

### 校验与反馈

- 实时校验：输入完成后 500ms 触发
- 提交校验：点击提交时全量校验
- 错误提示：字段下方红色文字，说明具体原因
- 错误字段自动滚动到可视区域并聚焦

### 表单状态

| 状态 | 表现 |
|---|---|
| `pristine` | 未修改，提交按钮默认可用 |
| `dirty` | 已修改，离开页面前提示保存 |
| `submitting` | 提交按钮 Loading，禁用其他操作 |
| `success` | Toast 提示成功，返回列表或继续 |
| `error` | 展示错误信息，保留表单数据 |

## 空状态

### 类型与表现

| 场景 | 图标 | 标题 | 描述 | 操作 |
|---|---|---|---|---|
| 无数据 | 📭 空盒 | 暂无数据 | 当前条件下没有记录 | [创建] |
| 搜索无结果 | 🔍 | 未找到结果 | 请尝试其他关键词或筛选条件 | [清除筛选] |
| 无权限 | 🔒 | 无权访问 | 请联系管理员申请权限 | [返回首页] |
| 无网络 | 📡 | 网络异常 | 请检查网络后重试 | [刷新] |
| 无通知 | 🔔 | 暂无通知 | 有新消息时会在这里显示 | - |

### 空状态结构

```text
┌─────────────────────────────┐
│                             │
│          [插画/图标]         │
│                             │
│         暂无患者             │  ← 标题，--text-lg
│      点击按钮添加第一位患者   │  ← 描述，--text-sm
│                             │
│      [+ 添加患者]            │  ← 操作按钮
│                             │
└─────────────────────────────┘
```

### 原则

- 空状态必须提供明确的下一步操作
- 避免仅展示「暂无数据」而无引导
- 图标/插画风格与整体设计一致

## 加载状态

### 初始加载

- 页面级：Skeleton 骨架屏
- 卡片级：卡片内部 Skeleton
- 列表级：列表项 Skeleton（3-5 条）

### 局部加载

- 按钮 Loading：Spinner + 文字变化
- 表格加载：表格行替换为 Skeleton，保留表头
- 下拉加载：下拉项区域 Skeleton

### 加载占位数量

| 场景 | 占位数量 |
|---|---|
| 卡片列表 | 4-6 张卡片 |
| 表格 | 5 行 |
| 消息列表 | 6 条 |
| 时间轴 | 3 个节点 |

### 加载超时与重试

- 超过 5 秒未加载完成，显示「加载较慢，请稍候」
- 加载失败切换到错误状态，提供重试按钮

## 错误状态

### 错误类型

| 类型 | 场景 | 表现 |
|---|---|---|
| 全局错误 | 应用崩溃、未捕获异常 | 错误页 + 刷新按钮 + 联系支持 |
| 页面错误 | 页面数据加载失败 | 页面内错误占位 + 重试 |
| 局部错误 | 卡片/列表加载失败 | 卡片内错误提示 + 重试 |
| 操作错误 | 提交失败、删除失败 | Toast 提示 + 保留表单状态 |
| 网络错误 | 请求超时、断网 | 提示网络异常 + 重试 |

### 错误提示结构

```text
┌─────────────────────────────┐
│      ⚠️ 加载失败             │
│   无法获取患者列表，请重试    │
│                             │
│      [重新加载]              │
└─────────────────────────────┘
```

### 错误信息规范

- 面向用户，避免技术术语（如 HTTP 500）
- 说明发生了什么 + 用户能做什么
- 提供明确的恢复操作（重试、返回、联系支持）

## 确认与撤销

### 破坏性操作确认

删除、移除、停用等操作必须二次确认：

```text
┌─────────────────────────────┐
│ 确认删除该患者？              │
│ 删除后数据将无法恢复。         │
├─────────────────────────────┤
│ [取消]         [删除]        │
└─────────────────────────────┘
```

### 撤销机制

- 删除后可撤销：Toast 显示「已删除 [撤销]」，持续 5 秒
- 状态修改后可撤销：重要状态变更提供撤销入口
- 批量操作后提供撤销

## 批量操作

### 批量选择

- 表格/列表首列提供 Checkbox
- 全选仅选中当前页可见项
- 选中后顶部出现批量操作栏

```text
┌────────────────────────────────────────────────────┐
│ 已选择 5 项        [批量完成] [批量分配] [删除] [×] │
└────────────────────────────────────────────────────┘
```

### 批量操作栏

- 固定在列表/表格上方
- 显示已选数量
- 操作按钮根据权限与选中项状态动态启用
- 点击「×」清空选择

## 数据刷新

### 自动刷新

- Dashboard 关键数据每 60 秒自动刷新
- 刷新时显示最后更新时间
- 避免整页刷新导致闪烁，使用局部更新

### 手动刷新

- 页面提供刷新按钮
- 列表/表格支持下拉刷新（移动端）
- 刷新失败保留旧数据并提示错误

### 最后更新时间

```text
┌────────────────────────────────────┐
│ 患者列表              更新于 09:42  │
└────────────────────────────────────┘
```

## 渐进展开

用于信息密度高的医疗场景，避免一次性展示过多内容。

### 使用场景

- 患者详情页 Timeline 事件展开更多指标
- Care Plan 卡片展开子任务
- 设置页面展开高级选项

### 模式

| 模式 | 说明 |
|---|---|
| `Accordion` | 折叠面板，一次可展开多项 |
| `Show More` | 长文本截断，点击展开剩余 |
| `Inline Expand` | 表格行内展开详情 |
| `Drawer` | 展开完整详情 |

### 原则

- 默认展示最关键信息
- 展开操作明确可预测
- 展开状态可记忆（可选）

## 通知与提醒

### 通知类型

| 类型 | 触发时机 | 展示方式 |
|---|---|---|
| 系统通知 | 风险升级、任务到期 | Toast + 通知中心 |
| 操作反馈 | 保存成功、删除完成 | Toast |
| 消息通知 | 收到新消息 | Badge + 通知中心 |
| 预警通知 | P1/P2 风险变化 | 置顶横幅 + Toast |

### 通知中心

- 入口在 Header 铃铛图标
- 未读消息显示 Badge
- 通知按时间倒序排列
- 支持标记已读、全部已读

## 模式使用速查

| 场景 | 推荐模式 |
|---|---|
| 查看详情但不跳转 | Drawer |
| 创建/编辑复杂对象 | 独立页面 |
| 删除/禁用等危险操作 | Modal 确认 |
| 筛选数据 | 顶部筛选栏 + Tag |
| 无数据 | 空状态 |
| 初次加载 | Skeleton |
| 操作完成 | Toast |
| 多选操作 | 批量操作栏 |
| 信息过多 | 渐进展开 |
