# PRD-20 验收标准

## 背景
多模块并行开发需要统一可执行验收标准。

## 为什么
验收标准缺失会导致“功能完成但不可上线”。

## 目标
定义功能、性能、安全、可观测性四类验收门槛。

## 非目标
- 不替代详细测试用例文档。

## 范围
覆盖 MVP 全模块上线前门禁。

## 流程图（Mermaid）
```mermaid
flowchart TD
  A[功能完成] --> B[测试通过]
  B --> C[安全与合规检查]
  C --> D[性能达标]
  D --> E[发布审批]
```

## ASCII 图
```text
Feature Done -> Test -> Security -> Perf -> Release
```

## 表格
| 类别 | 基线 |
|---|---|
| 功能 | 关键用户路径通过率 100% |
| 质量 | P0/P1 缺陷为 0 |
| 性能 | P95 API < 500ms（核心读接口） |
| 安全 | 高危漏洞为 0 |
